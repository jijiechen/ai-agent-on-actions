
请首先读取 `.agents/prompts/base.md` 中的基础知识和指令。

按照以下步骤开始你的工作：

# 1. 工作分类

1. 读取用户从 GitHub 的 issue 或 PR 的评论或回复，判断当前这次任务的类型，并在文件 `/tmp/copilot/task-kind.txt` 里输出任务类型。如果 issue 的评论内容只有 `/ai-agent` 而没有其他字样，这表示希望你创建 PR 来实现需求或优化现有已关联的 PR，也就是 dev 类型任务；如果是 issue 或 PR 的 label 事件，也是 dev 类型任务，其他的，请按下面的方式来判断：
   1. issue 文字整理工作或需求重写。输出任务类型为 issue_refinement。
   2. 涉及到需要修改仓库代码的工作，输出任务类型为 dev。包括要实现功能或自动化测试，修改 CI 或仓库里 文档都属于这一类。
   3. 涉及到要验证产品的某项产品功能，输出类型为 test。
   4. 日常杂务，输出任务类型为 generic。包括询问仓库统计, GitHub 项目管理，以及普通日常对话闲聊等各种其他类型的工作。
2. 无论是哪项工作，都请在原始 issue 或 PR 上打上 GitHub Reactions（表情反馈指示）
3. 如果是日常杂务 generic 类工作，你就自己完成它；其他类型的工作就直接跳过，不要执行任何操作，后面我会有其他程序负责执行的

# 2. 准备开始

**为所有类型的工作都添加 `eyes` 👀 反应**

你的任务包括向将所有类型的任务添加开始的 `eyes` 反应。请按照 `.agents/prompts/base.md` 中的说明完成添加。

# 3. 执行工作

如果是日常杂务 generic 类工作，你就自己完成它；其他类型的工作你直接跳过，不要执行任何操作，后面我会有其他程序负责执行。

再次强调，你只执行 generic 类工作。

# 4. 工作结束

如果是日常杂务 generic 类工作，是你自己做的，在结束时额外执行以下步骤：
1. 所有步骤完成后，把你的工作结果、结论更新到 issue 或 PR 上作为一条新的评论。请使用中文输出、写 PR 标题和评论。
2. 根据任务结果、结论，按照 `.agents/prompts/base.md` 中的说明完成添加对评论或 PR/Issue 的反应：如何任务正常完成，移除 `eyes`，不要添加任何其他反应。
