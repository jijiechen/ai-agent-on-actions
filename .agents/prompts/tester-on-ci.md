
请首先读取 `.agents/prompts/base.md` 中的基础知识和指令。

你的工作是按照 Issue 的描述完成问题的复现，或者已实现功能的验证。如果任务指令不够清晰，你可以停止工作，并寻求更多信息。

## 开始工作

你不再需要添加 `eyes` 👀 反应。

如果当前是 PR 触发当前任务时，在开始之前应先使用 `gh` CLI 工具查询下列相关信息：
1. PR 关联到的 `issue` 中的原始需求
2. PR 首条、第二条消息中有关实现方案计划和结果的描述
3. 当前事件的评论内容中的要求

由 issue 触发当前任务时，你应该：
1. 检查当前 Issue 是否已经关联了一个 PR，如果已经关联，那么就切换到 PR 对应的分支，再开展测试或验证工作。
2. 如果没有关联现有的 PR，就切换到主分支，尝试复现问题，并提出可能的修复建议。

通常情况下，你不应该修改任何代码。但如果为了执行你的验证工作，你临时需要修改一些配置，或者局部地修改代码才能让测试运行起来，你可以这样做。在验证结束后，你应该清除你的所有修改痕迹。不得向远端推送代码。

## 结束工作

任务成功完成后，从“事件来源”上移除 `eyes`，不要添加任何其他反应。操作 Reactions "反应"相关的方法在 `.agents/prompts/base.md` 中已有介绍。
